# PR 5707: [CBRD-25722] Enable correlated subquery caching with type_sp included in the subquery.

- state: MERGED / base: develop / created: 2024-12-12
- diff: +17 -2, 1 files

## Code changes (summary)

- `pt_make_sq_cache_key_struct` in `src/parser/xasl_generation.c`: added a new `case TYPE_SP:` branch that iterates `regu_src->value.sp_ptr->args` and recursively feeds each argument's regu var into the cache-key struct via `SQ_TYPE_REGU_VAR`, following the same iterate-and-accumulate pattern used for other multi-argument regu var types.
- Removed `TYPE_SP` from the fallback switch cases (alongside `TYPE_CLASSOID`, `TYPE_REGUVAL_LIST`, `TYPE_REGU_VAR_LIST`) that previously returned `ER_FAILED` with a "Result Cache not supported" comment.
- Net effect: correlated scalar subqueries whose select list includes a stored-procedure-style call (`TYPE_SP` regu var, e.g. system/stored procedure invocation) are now eligible for subquery result caching instead of being rejected outright.
