# PR 6166: [CBRD-25997] Skip end_one_iteration when it's possible in parallel heap scan

- state: MERGED / base: feature/parallel_query / created: 2025-05-12
- diff: +60 -53, 3 files

## Code changes (summary)

- `px_heap_scan_memory_mapper.cpp`/`.hpp`: change `add_resolved_dbval_all()` to return `bool` (whether all `vfetch_to` domains are now resolved) instead of `void`, and only clone/store a dbval when its domain is not null; handle both `REGU_VARIABLE` and `REGU_VARIABLE_LIST` map entries uniformly via a shared `clone` pointer, skipping unrelated map-entry types with `continue`.
- `set_all_regu_var_domain_refer_to_clone()`: only overwrite `orig->vfetch_to` from the resolved-value map when `orig->vfetch_to` is still `DB_IS_NULL`, avoiding clobbering an already-resolved value; refactored to share the same orig/clone extraction pattern.
- `px_heap_scan_task.cpp`: only mark `resolved_dbval_stored` when `add_resolved_dbval_all()` reports all domains resolved, instead of unconditionally setting it true after one call — fixes a restrack assertion caused by copying into a dbval that was never `clear()`-ed.
- Add `#include "dbtype.h"` to `px_heap_scan_memory_mapper.cpp` for the `DB_IS_NULL` macro.

## PR body (verbatim)

<http://jira.cubrid.org/browse/CBRD-25997>

### Purpose

#6091 PR 이후 restrack assertion이 발생하여 확인해본 결과, dbval을 덮어씌우기 전 clear를 수행하지 않은 코드를 발견하였습니다. 이를 수정합니다.
또한, null인 column이 많은 상황에서 유효한 dbval만 저장할 수 있도록 코드를 수정합니다.

### Implementation

NULL 인 경우에만 복사를 수행하게끔 변경
null이 아닌 dbval을 모두 저장할 수 있을 때 까지 domain resolve 용 dbvalue 복사를 시도하게끔 변경

### Remarks

> null이 아닌 dbval을 모두 저장할 수 있을 때 까지 domain resolve 용 dbvalue 복사를 시도하게끔 변경

이 때문에, group by aggregate가 포함된 쿼리를 null이 많은 table 대상으로 수행할 때 parallel heap scan 성능이 낮아질 위험이 있습니다. (계속 domain resolve를 시도하기 때문)
