## PR body (verbatim)

<http://jira.cubrid.org/browse/CBRD-26104>

### Purpose

Parallel hash join에서 parallel query 구현 및 수행 시 사용하는 공용 함수, 변수, 자료구조에 대한 변경이 많습니다. 이를 통해 유지보수성을 개선하였습니다.

Parallel subquery execution의 기존 코드는 여러 개의 XASL을 task로 만들어 cubthread의 worker pool task queue에 push하는 방식이었으나, push 과정에서 mutex 사용으로 성능 부하가 발생했습니다. 이를 개선하기 위해 단일 task 구조로 리팩토링하고, thread-safe job queue를 구현하여 성능 저하 없이 push가 가능하도록 수정하였습니다.

이후 parallel subquery execution을 단일 task 구조로 전환한 상태에서 parallel heap scan을 수행할 경우, worker_manager가 하나의 스레드에서 하나의 parallel query processing만 허용하는 문제가 있어 이에 대한 리팩토링을 추가로 진행했습니다.

또한 m_qlist_count나 m_px_lock(IS_LOCK을 위한 lock) 설정 시 실제 쿼리를 실행하는 main thread까지 재귀적으로 탐색하지 않으면 parallel query thread가 parallel heap scan 등을 수행할 때 오류가 발생하는 문제를 수정했습니다.

Query profiling의 경우, tran_index 단위로 정보를 수집하는데 자료형이 int라 thread-safe하지 않았습니다. 이를 각 thread별(m_px_stats)로 trace 정보를 수집하고, parent thread(병렬 실행 thread일 수 있음)에서 병합한 뒤 최종적으로 main thread에 trace 정보를 모을 수 있도록 재귀적 병합 구조로 리팩토링했습니다.

마지막으로 xasl_iteration과 xasl→header.id를 구현하여 xasl tree 상에서 id 기반으로 원하는 XASL을 탐색할 수 있도록 하였으며, parallel query thread가 xasl clone에서 특정 XASL에 접근할 수 있게 리팩토링했습니다.
