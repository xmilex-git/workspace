# PR 6877: [CBRD-26574] Enhance memoize performance

- state: MERGED / base: develop / created: 2026-02-27
- diff: +9 -6, 2 files

## Code changes (summary)

- `src/query/memoize.cpp`: added a `clear_memoize_storage (thread_p, xasl)` call in the three branches where memoize is being disabled (after `stop_timer()`), so the memoize storage is torn down immediately instead of staying allocated and continuing to add overhead after being marked inactive.
- `src/query/parallel/px_heap_scan/px_heap_scan.cpp`: reordered cleanup in the task-finalization path — the "release worker manager" (`m_worker_manager->wait_workers()`) block was moved earlier, right after `read_finalize`, so workers are joined before the result handler is torn down instead of after.
- Net effect: once the 1000-probe / <50%-hit-rate condition triggers memoize disablement, the per-query memoize state is reset right away, removing the residual performance penalty that previously persisted after disablement.
