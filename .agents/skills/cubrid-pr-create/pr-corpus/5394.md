## Code changes (summary)

- `heap_scan_get_visible_version` in `src/storage/heap_file.c`: restructured the visibility-check logic so it branches first on whether `class_oid != NULL`.
- Previously, when `class_oid` was `NULL`, the combined `need_check_visibility` boolean short-circuited to false and the MVCC snapshot check was skipped entirely, returning the peeked record directly.
- Now, when `class_oid != NULL`, the code still checks `mvcc_is_mvcc_disabled_class` and, if MVCC is enabled for the class, evaluates the snapshot function and only returns early on `SNAPSHOT_SATISFIED`.
- When `class_oid == NULL`, the new `else` branch explicitly treats it as an "mvcc_disabled_class" case and returns the peeked record directly (equivalent old shortcut), but this path is now reached deliberately rather than as a side effect of the removed combined condition.
- Net effect: rows with `class_oid == NULL` (e.g., user connection/access status info) that are not for MVCC-disabled classes now correctly go through MVCC snapshot visibility checks instead of always skipping them, fixing the wrong access-status print regression.
