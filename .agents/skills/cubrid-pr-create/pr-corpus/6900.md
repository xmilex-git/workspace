## Code changes (summary)

- `src/query/parallel/px_heap_scan/px_heap_scan.cpp`: before destroying `m_result_handler`, now calls `m_result_handler->read_finalize (m_thread_p)` so the handler is properly finalized (e.g. `qlist_count` bookkeeping) prior to its destructor and `db_private_free` being invoked.
- `src/query/parallel/px_heap_scan/px_heap_scan_checker.cpp`: for `TYPE_ATTR_ID` / `TYPE_SHARED_ATTR_ID` / `TYPE_CLASS_ATTR_ID` predicate args, now sets `CANNOT_PARALLEL_HEAP_SCAN` when the attribute's declared type is `DB_TYPE_OBJECT` or `DB_TYPE_OID`, excluding such queries from the parallel heap scan path.
- Together these fix a core dump previously reported in `qexec_execute_query` (`query_executor.c:16449`), which was linked to `qlist_count`/result-handler state not being finalized correctly and to object/OID-typed attribute predicates being incorrectly allowed through parallel heap scan.
