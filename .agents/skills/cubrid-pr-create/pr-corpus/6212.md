### Purpose

기존 CUBRID 소스에서는 holdable query 수행 시 qentry_p->list_id->page_cnt 값이 0이면, temp file을 생성하지 않은 쿼리로 간주하여 해당 temp file을 preserve하지 않았습니다. 
그러나 parallel heap scan 도입 이후에는 list_id에 직접 연결되어 있지 않더라도 결과를 포함하는 temp file이 존재할 수 있으며, 이러한 파일도 해당 qentry와 함께 관리되어야 합니다. 
이에 따라 qentry_p->list_id->page_cnt가 0일 때 temp file preserve를 포기하던 로직을 제거하였습니다.

In the original CUBRID source, during the execution of a holdable query, if the value of qentry_p->list_id->page_cnt was 0, the query was considered to have not created a temp file, and thus the corresponding temp file was not preserved.

However, with the introduction of parallel heap scan, a temp file containing results may exist even if it is not directly linked to the list_id. Such files also need to be managed along with the corresponding qentry.

Accordingly, the logic that skipped preserving the temp file when qentry_p->list_id->page_cnt was 0 has been removed.
