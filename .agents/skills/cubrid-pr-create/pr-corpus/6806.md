# PR 6806: [CBRD-26522] Support NL join during parallel heap scan

- state: MERGED / base: develop / created: 2026-01-28
- diff: +1258 -315, 28 files

## Code changes (summary)

- Added a new `NLJ_KEEP_HEAP_PAGE_PINNED` query hint (grammar, `parse_tree.h`, `parse_tree_cl.c` printer, `scanner_support.c`, `xasl_generation.c`) that forces `ACCESS_SPEC_FLAG_FORCE_FIXED_SCAN` on the driving spec so its heap page stays fixed during the join.
- New files `px_heap_scan_join_info.{cpp,hpp}`: `join_info` class that captures, records, and applies per-scan-node state (OID/HFID/BTID/list_id/status/qualified_block) keyed by XASL node id, shared across parallel worker threads under a mutex.
- `px_heap_scan_task::init` now walks the full `scan_ptr` chain (not just the driving heap scan) and opens heap/index/list scans for every join level inside each worker thread, instead of only opening the outer driving scan.
- `px_heap_scan_task::loop`: for `MERGEABLE_LIST`/`COUNT_DISTINCT` results, each worker now drives the nested-loop join's inner `scan_ptr` chain per outer tuple via a new `qexec_execute_scan_ptr`, writing joined rows directly from the worker instead of funneling rows through a central/serial join step.
- `xasl_state` is now allocated per-task (`m_xasl_state`) rather than shared, avoiding cross-thread mutation of `val_descr`/`query_id`/`qp_xasl_line`.
- `query_executor.c`: moved the "final" value-clearing block in `qexec_clear_xasl` to run after `scan_end_scan`/`scan_close_scan`; added `qexec_execute_scan_ptr` and `qexec_clear_head_lists_with_truncate` helpers; `qexec_open_scan`/`qexec_execute_mainblock_internal` updated to honor the forced-fixed-scan flag and to pass the correct per-node xasl.
- `list_file.c`, `query_manager.c/h`, `scan_manager.c/h`, `query_list.h`, `xasl.h`, `xasl_iteration.cpp`: supporting plumbing (null-safety checks, new flags/fields, list-truncation-while-open helpers) to enable the new per-thread NL-join execution path.
- Trace/stat handling reworked: sibling-XASL stats are now merged via `m_trace_handler->m_trace_storage_for_sibling_xasl.merge_xasl_tree` instead of a mutex-guarded loop over `dptr_list`.

## PR body (verbatim)

<http://jira.cubrid.org/browse/CBRD-26522>

### Purpose
기존에는 join driving 테이블이 heap scan인 경우, 이를 row-by-row로 처리하여 동기화 오버헤드 및 중앙 처리 방식으로 인해 NL join의 성능이 많이 저하됐었습니다.

이를 개선하기 위해, 각각의 parallel heap scan thread에서 NL join을 평가하고 결과를 각각 수집하게끔 변경하여, mergeable-list 방식으로 처리할 수 있도록 변경합니다.
