## Code changes (summary)

- In `file_tempcache_unlock_tran_entry` (src/storage/file_manager.c), reordered the debug-only `owner_mutex = -1` reset to happen before `pthread_mutex_unlock`, instead of after.
- Fixes a thread-safety race where, after another thread could acquire the mutex, the current thread still wrote to `owner_mutex`, corrupting the debug ownership tracking.
