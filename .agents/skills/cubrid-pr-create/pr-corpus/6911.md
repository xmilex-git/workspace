# PR 6911: [CBRD-26615] Reduce I/O bottleneck when parallel heap scan

- state: MERGED / base: develop / created: 2026-03-17
- diff: +515 -229, 11 files

## Code changes (summary)

- `src/storage/file_manager.c/h`: added `file_get_all_data_sectors()`, which reads the heap file header once and walks both the PART_FTAB (partially-used) and FULL_FTAB (full) sector tables to collect every data-bearing `FILE_PARTIAL_SECTOR` (VSID + page bitmap) into a `FILE_FTAB_COLLECTOR`; moved `FILE_ALLOC_BITMAP`/`FILE_PARTIAL_SECTOR`/`FILE_FTAB_COLLECTOR` type definitions from the `.c` file into the public header so callers outside file_manager can use them.
- New file `px_heap_scan_ftab_set.hpp`: `ftab_set` container/iterator around a `std::vector<FILE_PARTIAL_SECTOR>`, with `convert()` (from a `FILE_FTAB_COLLECTOR`), `split(n)` (partition sectors evenly across `n` worker sets), `get_next()`, and `clear()`.
- Replaced `input_handler_single_table` (renamed/rewritten as `input_handler_ftabs`, `.hpp`/`.cpp`) and deleted the old abstract `input_handler` base class: the old design serialized page allocation through a single global-mutex `page_next` call per worker; the new design's `init_on_main` fetches all data sectors once via `file_get_all_data_sectors` and splits them across workers up front (`ftab_set::split`), while `initialize`/`get_next_vpid_with_fix` let each worker thread iterate its own pre-assigned sector range and `pgbuf_ordered_fix` pages independently, without any shared lock.
- `px_heap_scan.cpp/.hpp` and `px_heap_scan_task.hpp` updated to use `input_handler` / `input_handler_ftabs` instead of `input_handler_single_table`, and `px_heap_scan::init` now calls `m_input_handler->init_on_main(...)` before starting workers, failing fast (cleanup + `ER_FAILED`) if it errors.
- `cubrid/CMakeLists.txt`: swapped the old `px_heap_scan_input_handler_single_table.{cpp,hpp}` build entries for `px_heap_scan_input_handler_ftabs.{cpp,hpp}` and the new `px_heap_scan_ftab_set.hpp` header.
- Net effect: parallel heap scan workers no longer synchronize per-page through a global mutex; each thread independently iterates its own pre-partitioned sector/page range, allowing I/O to proceed concurrently across threads instead of being serialized at the point of page allocation.

## PR body (verbatim)

<http://jira.cubrid.org/browse/CBRD-26615>

### Purpose

Parallel Heap Scan 수행 시 발생하는 I/O 병목 현상을 해소하기 위해 스캔 분배 방식을 최적화합니다.

기존 방식은 전역 Mutex를 사용하여 page_next 로직을 통해 페이지를 하나씩 순차적으로 할당받는 구조였습니다. 이로 인해 스레드가 페이지를 할당받고 I/O를 수행하는 과정에서 동기화 호출이 잦아지고, 사실상 I/O를 기다리는 시점이 직렬화되어 다중 스레드의 이점을 충분히 활용하지 못하는 병목이 존재했습니다.

이를 개선하기 위해, 스캔 시작 단계에서 Heap File의 헤더를 한 번만 읽어 전체 데이터가 포함된 Partial Sector (64페이지 단위) 정보를 미리 수집(Fetch)합니다. 수집된 섹션 정보들을 워커 스레드들에게 미리 나누어 할당함으로써, 각 스레드가 자신에게 할당된 섹터 범위를 독립적으로 순회(Iteration)하며 별도의 락 없이 병렬적으로 I/O를 발생시키고 데이터를 처리할 수 있도록 구조를 개선합니다.

### Implementation

Storage - File Manager (src/storage/)

`file_manager.c`: file_get_all_data_sectors 함수를 추가하여 힙 파일의 PART_FTAB(일부 사용 중인 섹터)과 FULL_FTAB(꽉 찬 섹터) 정보를 모두 순회하며 실제 데이터가 포함된 섹터 정보를 FILE_FTAB_COLLECTOR로 수집하도록 구현했습니다.
`file_manager.h`: 섹터 수집을 위한 구조체 및 비트맵 매크로(FILE_FULL_PAGE_BITMAP 등)와 외부 인터페이스를 선언했습니다.

`px_heap_scan_input_handler_ftabs.cpp:`
init_on_main: 메인 스레드에서 file_get_all_data_sectors를 호출하여 전체 섹션 위치를 파악하고 병렬도에 맞춰 분할합니다.
get_next_vpid_with_fix: 할당된 섹터 내의 비트맵을 로컬 스레드에서 직접 확인하며 페이지를 pgbuf_fix 하도록 수정하여, 전역 Mutex 없이도 독립적인 I/O 발생이 가능하도록 로직을 변경했습니다.
