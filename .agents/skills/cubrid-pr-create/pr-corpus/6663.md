## Code changes (summary)

- `xasl_generation.c`의 `PT_EXISTS` 처리: `pt_prepare_corr_subquery_hash_result_cache()`가 성공하면 해당 xasl에 `XASL_USES_SQ_CACHE` 플래그를 설정해 exists 서브쿼리도 캐시 대상이 되도록 함.
- `query_evaluator.c`의 `eval_pred()`: `TYPE_LIST_ID` 경로에서 캐시 사용 여부(`XASL_USES_SQ_CACHE`)를 확인해 `sq_get()`으로 캐시 히트를 조회하고, 미스 시 실행 후 `sq_make_key()`/`sq_put()`으로 캐시에 저장하는 분기를 추가; 캐시를 쓰지 않는 경우는 기존 실행 경로 유지.
- `subquery_cache.h`: `sq_regu_value` 유니온의 `srlist_id` 필드를 `bool exists`로 교체해, list_id 전체 대신 exists 여부만 저장하도록 단순화.
- `subquery_cache.c`: `sq_make_val()`에서 `TYPE_LIST_ID`를 `tuple_cnt > 0` 기반 `exists` bool로 변환; `sq_unpack_val()`에서 list_id 복제 대신 `tuple_cnt`를 0/1로 설정; `sq_put()`의 엔트리 크기 계산에 `TYPE_LIST_ID` case 추가.
- `sq_hash_func()`: 해시 계산을 `mht_valhash` 누적 XOR에서 `ROTL32` + `mht_get_hash_number` 조합으로 변경해 해시 품질 개선.
- `sq_get()`의 캐시 축출 조건을 `SQ_CACHE_MISS` 기반 임계값에서 `SQ_CACHE_SIZE`가 `SQ_CACHE_SIZE_MAX`의 60%를 넘는지로 변경.

## PR body (verbatim)

<http://jira.cubrid.org/browse/CBRD-26400>

### Purpose

기존의 correlated subquery cache는 scalar subquery에 대해서만 지원하였는데, exist문에 대해서도 지원 가능하게끔 변경하려고 합니다.

### Implementation

subquery_cache.c 및 xasl_generation을 수정하여 regu_var (type:srlist_id) 에 대한 subquery cache 지원 추가
