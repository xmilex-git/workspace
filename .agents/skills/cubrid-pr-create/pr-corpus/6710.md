# PR 6710: [CBRD-26431] COUNT(alias) 사용 시, 서브쿼리 표현식이 재작성되며 결과가 0으로 계산되는 문제

- state: MERGED / base: develop / created: 2025-12-15
- diff: +29 -3, 1 files

## Code changes (summary)

- `px_heap_scan_result_handler.cpp`에 `fetch.h` include 추가.
- group-by가 있는 count aggregate 처리 루프에서, operand가 `TYPE_CONSTANT`가 아닌 경우를 위한 분기 추가: 기존에는 `assert(operand->value.type == TYPE_CONSTANT)`로 상수만 가정했으나, 이제 상수가 아니면 `fetch_peek_dbval()`로 실제 값을 평가해 `db_value_p`를 얻고 실패 시 `false` 반환.
- group-by가 없는(단일) count aggregate 처리 분기에서도 동일하게 `agg_node->operands->value.type`이 `TYPE_CONSTANT`인지 확인 후, 아니면 `fetch_peek_dbval()`로 값을 평가하도록 수정.
- 목적: count(alias)처럼 서브쿼리/표현식으로 재작성된 operand가 상수가 아닌 경우에도 parallel heap scan count 최적화 경로에서 값을 올바르게 평가해 결과가 0으로 잘못 계산되는 문제를 해결.
