- state: CLOSED / base: develop / created: 2025-11-28
- diff: +466 -3, 8 files

## Code changes (summary)

- `page_buffer.c/.h`에 스레드-로컬 "cached fix" 메커니즘을 신규 도입: `pgbuf_Cache` 전역 구조체, 스레드별 `PGBUF_CACHE_ARRAY`/`PGBUF_CACHE_ENTRY`를 두어 자주 접근하는 페이지의 fake BCB/iopage 사본을 캐싱.
- `pgbuf_cache_global_initialize/finalize`, `pgbuf_cache_initialize/finalize`, `pgbuf_cache_entry`, `pgbuf_cached_fix`, `pgbuf_cached_unfix`, `pgbuf_unfix_without_cache` 등 신규 함수 추가; `pgbuf_bcb`에 `write_seq`(atomic) 및 `is_cached_fake_bcb` 필드 추가해 캐시된 BCB의 변경 여부(쓰기 시퀀스)를 검증.
- 페이지에 write latch가 걸리거나 flush될 때(`pgbuf_latch_bcb_upon_fix`, `pgbuf_bcb_flush_with_wal`, `pgbuf_promote_read_latch_release` 등)마다 `write_seq`를 증가시켜 캐시 무효화 트리거로 사용.
- `pgbuf_unfix()`에서 캐시된 fake BCB인 경우 실제 unlatch 없이 바로 리턴하도록 우회 경로 추가.
- `btree.c`의 root/child 페이지 fix를 `pgbuf_fix` 대신 `pgbuf_cached_fix`로 교체해 B-tree 탐색 시 캐시 활용.
- `scan_manager.c`의 `scan_open_index_scan()`에서 `pgbuf_cache_initialize()` 호출 추가; `query_executor.c`에서 update/delete/insert/merge 시작 시 `m_enable_cached_fix = false`, 순수 조회 쿼리(`qexec_execute_query`) 동안만 `true`로 설정해 캐시를 읽기 전용 시나리오로 제한.
- `thread_entry.hpp/.cpp`, `thread_entry_task.cpp`에 스레드 컨텍스트별 캐시 상태 필드(`m_enable_cached_fix`, `m_pgbuf_cache_count/max/entries`) 및 스레드 재사용 시 `pgbuf_cache_finalize()` 호출 추가.
- PR body가 N/A로 비어 있고 상태가 CLOSED인 것으로 보아 실험적/미완성 변경(동일 VPID 동시 접근에 의한 data buffer 병목 완화 시도)으로 추정.
