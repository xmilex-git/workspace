## Code changes (summary)

- `qdump_print_stats_json()` (`query_dump.c`): the `UNION_PROC`/`DIFFERENCE_PROC`/`INTERSECTION_PROC` case now emits `time`/`fetch`/`fetch_time`/`ioread` stats for the node itself (previously it only recursed into `proc.union_.left`/`right` with no stats), and additionally iterates `xasl_p->aptr_list` to emit a `"SUBQUERY (uncorrelated)"` JSON array so CTE/UNION-bundled subqueries are traced
- `aptr_list`/`dptr_list` subquery reporting changed from a single JSON object to a JSON array built by iterating the linked list, so multiple subqueries bundled under one node (e.g. under a UNION) are all output instead of just the first/head one
- Added `fetch_time` stat output for the `BUILD_SCHEMA_PROC` case
- `qdump_print_stats_text()`: merged the previously-separate `UNION`/`DIFFERENCE`/`INTERSECTION` text branches into one shared branch that prints per-node stats (time/fetch/fetch_time/ioread) and loops over `aptr_list`; `aptr_list`/`dptr_list` printing likewise changed from a single recursive call to a loop over the full list

## PR body (verbatim)

http://jira.cubrid.org/browse/CBRD-25361

There are currently issues with trace statistics. When tracing queries that use UNION, DIFFERENCE, INTERSECTION, and CTE together, the statistical information for the CTE is not output. Additionally, when using UNION-like SQL statements, the trace for the total cost is also not output. In this issue, we have made changes to enable the output of trace for the total cost of UNION-like SQL statements and to ensure that the statistical information for all subqueries bundled under the UNION statement is output.
