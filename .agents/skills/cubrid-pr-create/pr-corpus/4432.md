# PR 4432: [CBRD-24850] Enhancing group_concat() Performance by Optimizing String Size and Length Calculation Methods

- state: MERGED / base: develop / created: 2023-06-19
- diff: +47 -16, 5 files

## Code changes (summary)

- Added a `medium.length` field to DB_CHAR union in DB_VALUE (`dbtype_def.h`) to cache string char-length, documented as "Only Use for group_concat() now"
- Initialize the new `length` field to -1 in `db_value_domain_min`/`db_value_domain_max`/`db_value_domain_default` (`db_macro.c`) and in `db_make_db_char` (`dbtype_function.i`)
- Propagate `length` when copying DB_VALUEs in `mr_setval_string`/`mr_setval_char`/`mr_setval_nchar`/`mr_setval_varnchar` (`object_primitive.c`)
- `db_get_string_length()` now returns the cached length directly when it is valid (`!= -1`) instead of always calling `intl_char_count()`
- `qstr_concatenate()` signature extended with `s1_size`/`s2_size` parameters so callers can pass precomputed sizes instead of having the function recompute them via `intl_char_size()`
- `db_string_concatenate()` passes precomputed length/size to `qstr_concatenate()` and stores the resulting length into `result->data.ch.medium.length`
- Removed now-redundant `intl_char_size()` calls inside `qstr_concatenate()` at multiple points (fixed-length branch, copy/cat length computation, final result size)

## PR body (verbatim)

http://jira.cubrid.org/browse/CBRD-24850

This pull request introduces the following changes to enhance the performance of the group_concat() function:

1. A **_medium.length_** component has been added to DB_CHAR, which is one of the constituents of DB_VALUE (db_value->data.ch.medium.length). Code has been added to **initialize this component to -1** when executing db_value_domain_default() and db_make_db_char() functions. (src/compat/dbtype_def.h, src/compat/db_macro.c, src/compat/dbtype_function.i)
2. When copying DB_VALUE, the pr_clone_value() function is executed, which in turn carries out mr_setval_string() on the destination. The code has been modified to also copy the **_length_** value during this process. (src/object/object_primitive.c)
3. A code modification has been included that checks if a **_length_** value is valid when executing the db_get_string_length() function. If the **_length_** value is valid, it is used. If not, a value obtained through intl_char_count() is stored in **_length_**.  (src/query/string_opfunc.c)
4. The code within qstr_concatenate has been updated to minimize the execution of the intl_char_count() and intl_char_size() functions. Instead, these values are now pre-received as arguments. (src/query/string_opfunc.c)
