## PR body (verbatim)

<http://jira.cubrid.org/browse/CBRD-26465>

### Purpose

1. 특정 조건에서 XASL clone과 xasl_unpack_info_ptr가 모두 존재하지 않아 parallel heap scan open()이 실패하며, 이 과정에서 worker_manager_p가 double free 되는 문제를 수정함.
2. 쿼리 실행 중 에러 발생 시, task가 종료·정리되기 전에 main 쿼리 스레드가 먼저 종료되어 task 종료 시 query id 등이 유실되며 오류가 발생하는 문제를 수정함. (task에 signal 후 spin wait 방식 적용)
3. PHS checker 리팩토링: 기존에는 count(*) 등이 row-by-row로 처리되었으나 count optimization 적용 이후 count fast path를 타게 되었고, 이로 인해 조건절에 rownum이나 Java SP 등이 포함된 경우에도 동일 경로로 처리되어 error/hang이 발생하던 문제를 해결함.
4. insert into t values (select …)와 같이 values 절에 scalar subquery가 포함된 경우, 해당 부질의에서 parallel heap scan이 처리되지 않고 invalid xasl content 에러를 반환하던 현상을 수정함. (xasl_iteration.hpp 수정)
5. order by 절이 존재하고 group by aggregate는 없으며, select list의 expr에 host variable이 포함된 경우 domain coerce가 수행되지 않아 order by가 정상 동작하지 않던 문제를 수정함. task에서 val_list를 채운 뒤 main thread에서 outptr-list에 대해 fetch_val_list를 1회 호출하여 db_variable 등으로 설정된 타입을 적절히 변환하도록 함.
6. Mergeable list에서 outptrlist에 값이 존재한 상태로 next()를 한 번 더 호출할 경우, private heap id를 0으로 변경하고 기존 값을 clear하면서 오류가 발생하던 문제를 수정함. next() 호출 시 task 실행 전에 한 번 clear()하도록 변경함.
7. 세션 변수 및 serial 사용 시 parallel heap scan을 사용할 수 없도록 변경함. (checker 리팩토링 이후 해당 기능이 유실되어 오류를 유발하던 문제 수정)
