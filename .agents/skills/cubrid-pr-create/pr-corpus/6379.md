# PR 6379: [CBRD-26244] __builtin_ 함수를 응용한 parallel query 성능 최적화

- state: MERGED / base: develop / created: 2025-08-18

## Code changes (summary)

- `fetch_val_list` (fetch.c): added `likely`/`unlikely` macros; wrapped the set-type check with `unlikely` and added a separate `DB_NEED_CLEAR` check to reduce the clear-call bottleneck for non-set values.
- `qfile_generate_tuple_into_list` (list_file.c): for `T_NORMAL` tuple type, calls `qfile_save_normal_tuple` directly instead of going through the generic `qfile_save_tuple` dispatch, simplifying the hot path.
- Mergable-list writer / `px_heap_scan_list_stream.cpp`: writes non-overflow tuples directly to the list_file temp page instead of building into a `tpl_buf` and then memcpy'ing; added `likely`/`unlikely` hints around `QPROC_TPLDESCR_SUCCESS`/`FAILURE`/`RETRY_*` branches.
- `scan_init_filter_info` and related scan-manager code (scan_manager.c/.h, query_evaluator.h, query_reevaluation.cpp, query_dump.c): inlined/unpacked initialization instead of a function call to reduce call overhead.
- `qdata_copy_valptr_list_to_tuple` / `qdata_generate_tuple_desc_for_valptr_list` (query_opfunc.c): added `unlikely`-hinted hidden-column branch checks with `continue`, and `unlikely`-hinted set-type check for `QPROC_TPLDESCR_RETRY_SET_TYPE`.
- `heap_file.c`: added `likely`/`unlikely` hints to `IS_DEDUPLICATE_KEY_ATTR_ID` and class-representation cache-validity checks; `heap_scan_get_visible_version`-related path returns early when `mvcc_header` is valid and all-visible, avoiding extra `scan_cache` memory access.
- Removed the `#if SERVER_MODE && !WINDOWS` guards from px_heap_scan_context/list_stream .cpp/.hpp files; reorganized `sa/CMakeLists.txt` to drop the standalone `PARALLEL_QUERY_SOURCES/HEADERS` and several now-unbuilt px_heap_scan/px_query_execute source files, shrinking non-SERVER_MODE binaries by excluding parallel heap scan (~1MB size reduction).

## PR body (verbatim)

<http://jira.cubrid.org/browse/CBRD-26244>

### Purpose

Parallel query 기능 중 Parallel heap scan과 Parallel subquery execution을 하나의 쿼리에서 동시에 사용하는 경우, 할당된 parallel workers 수만큼의 성능 향상이 발생하지 않고 병목이 관찰됩니다.
이를 개선하기 위해 성능 프로파일링을 수행하였으며, 주요 병목 지점을 최적화하여 병렬 실행 효율을 향상시켰습니다.

### Implementation

- fetch_val_list 최적화
  - db_value의 type이 set인 경우 clear 호출에서 병목 발생.
  - set이 아닌 경우를 기본 가정으로 하는 매크로 삽입 및 __builtin_expect 활용으로 branch misprediction 방지.
- heap scan의 regu_var 배열 clear 루틴 변경
  - row 스캔 전 regu_var 내부 db_value need_clear 검사에서 memory stall 발생.
  - row fetch 시점에서 clear 여부 검사 및 수행으로 변경.
- qfile_save_tuple 분기 단순화
  - 함수 진입부에서 부하 발생.
  - T_NORMAL의 경우 qfile_save_normal_tuple을 직접 호출하도록 수정.
- mergable list writer memcpy 최적화
  - 기존: tpl_buf에 tuple 생성 후 memcpy.
  - 개선: overflow tuple이 아닌 경우 직접 list_file temp page에 쓰도록 변경.
- QPROC_TPLDESCR_SUCCESS 예측 최적화
  - overflow tuple이 발생하지 않는 경우를 기본 가정하여 __builtin_expect 적용.
- scan_init_filter_info 최적화
  - 함수 호출 시 부하 발생.
  - 함수를 unpack하여 직접 변수 초기화 수행.
- qdata_copy_valptr_list_to_tuple 분기 최적화
  - hidden column 여부 분기에서 branch misprediction 발생.
  - __builtin_expect 및 continue 추가로 예측 정확도 개선.
  - 동일 최적화를 qdata_generate_tuple_desc_for_valptr_list에도 적용.
- DEDUPLICATE_KEY_ATTR_ID 체크 최적화
  - 보통은 해당되지 않는 경우가 많으므로 false 예측을 기본 가정하도록 수정.
- heap_scan_get_visible_version 최적화
  - scan_cache 접근 시 memory stall 발생.
  - mvcc_header가 유효하고 all_visible 상태일 경우 즉시 반환하도록 수정.
- 빌드 설정 정리
  - CMakeLists 수정 및 SERVER_MODE 제거.
  - SERVER_MODE가 아닌 바이너리에는 parallel heap scan 기능 제외하여 바이너리 크기 축소. (약 1MB 감소)
