# PR 5623: [CBRD-25680] When using a correlated subquery that includes columns from an outer table in the select clause, it is not selected as a cache key

- state: MERGED / base: develop / created: 2024-11-13
- diff: +58 -2, 3 files

## Code changes (summary)

- `src/query/regu_var.hpp`: added a new flag constant `REGU_VARIABLE_CORRELATED` (`0x800`) for marking a regu variable as "for correlated scalar subquery cache".
- `src/parser/xasl_generation.c` (`pt_attribute_to_regu`): when an attribute resolves to a correlated variable from an enclosing scope, it is now tagged with `REGU_VARIABLE_SET_FLAG (regu, REGU_VARIABLE_CORRELATED)`.
- `pt_make_sq_cache_key_struct`: added a `regu_var_list_p` local and new logic to walk `xasl_src->outptr_list` (and, for `BUILDVALUE_PROC`/`BUILDLIST_PROC`, the aggregate operand list) and feed each regu var into the cache-key struct via recursive `SQ_TYPE_REGU_VAR` calls вАФ previously only `pred` was used to build the cache key, so outer-table columns referenced only in the select list were not part of the key.
- `pt_make_sq_cache_key_struct` (`TYPE_CONSTANT` case): guarded so a constant regu var is only folded into the cache key when it carries the new `REGU_VARIABLE_CORRELATED` flag, preventing outer-join constant regu vars (used internally by xasl outptr_list) from being mistakenly treated as cache-key-worthy outer columns.
- `src/parser/view_transform.c` (`mq_rewrite_aggregate_as_derived`): extended the hint bitmask handling so `PT_HINT_NO_SUBQUERY_CACHE` (in addition to `PT_HINT_NO_MERGE`/`PT_HINT_QUERY_CACHE`) is preserved on the aggregate select and excluded from the derived subquery when rewriting aggregates as derived tables.
- Net effect: subquery result caching now correctly incorporates outer-table columns referenced in the SELECT list as part of the cache key, fixing spurious cache hits, while avoiding false-positive cache-key inclusion from join-related constant regu vars.

When using a correlated subquery that includes columns from an outer table in the select clause, the subquery cache does not use those columns as keys, resulting in improper caching. Consequently, cache hits occur even in situations where cache misses are expected. To address this, we modified the code to include regu_var from xasl outptr_list as part of the caching key. Additionally, unlike pred, xasl outptr_list uses the outer table as a constant regu_var during joins, which could mistakenly utilize even non-outer columns as cache keys. Therefore, we adjusted the code to only enable caching when there is no table from the subquery that matches the specification.
