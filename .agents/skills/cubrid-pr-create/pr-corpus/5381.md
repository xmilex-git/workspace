# PR 5381: [CBRD-25361] Fixed the issue where trace information for CTEs and subqueries not provided. (#5200)

## Code changes (summary)

- `qdump_print_stats_json`: for `UNION_PROC`/`DIFFERENCE_PROC`/`INTERSECTION_PROC`, replaced the old `left`/`right` pair (which read `xasl_p->proc.union_.left/right`) with a loop over `xasl_p->aptr_list`, emitting each subquery into a `"SUBQUERY (uncorrelated)"` JSON array.
- `qdump_print_stats_json`: the generic `HAVE_SUBQUERY_PROC` uncorrelated-subquery branch was rewritten to always build a JSON array from the full `aptr_list` chain instead of special-casing a single subquery vs. a nested-subquery object.
- `qdump_print_stats_json`: the correlated `dptr_list` branch likewise switched from a single JSON object to a JSON array covering every entry in the `dptr_list` chain.
- `qdump_print_stats_text`: `UNION_PROC`/`DIFFERENCE_PROC`/`INTERSECTION_PROC` cases now share one code path that prints the XASL type name plus timing/fetch/ioread stats and iterates `aptr_list`, instead of three near-duplicate blocks calling into `proc.union_.left/right`.
- `qdump_print_stats_text`: uncorrelated and correlated subquery text dumps were changed to loop over `aptr_list`/`dptr_list` respectively rather than dumping only the head node.
- Net effect: trace statistics for CTEs and subqueries bundled under UNION-like SQL statements are now fully output, including total cost for UNION-like statements, addressing the previously missing/partial trace output.

## PR body (verbatim)

http://jira.cubrid.org/browse/CBRD-25361

There are currently issues with trace statistics. When tracing queries that use UNION, DIFFERENCE, INTERSECTION, and CTE together, the statistical information for the CTE is not output. Additionally, when using UNION-like SQL statements, the trace for the total cost is also not output. In this issue, we have made changes to enable the output of trace for the total cost of UNION-like SQL statements and to ensure that the statistical information for all subqueries bundled under the UNION statement is output.
