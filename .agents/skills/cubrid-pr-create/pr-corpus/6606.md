## PR body (verbatim)

<http://jira.cubrid.org/browse/CBRD-26370>

### Purpose

Parallel heap scan에서 COUNT(*), COUNT(column) 집계 연산에 대한 최적화를 구현했습니다.
기존에는 일반 row-by-row 방식으로 처리하던 count 쿼리를 병렬 처리하여 성능을 향상시켰습니다.
특히 UPDATE STATISTICS 작업에서 사용되는 count distinct 연산을 병렬로 처리할 수 있도록 개선했습니다.

### Implementation

1. 새로운 Result Type 추가
px_heap_scan_result_type.hpp Line 3
    COUNT_DISTINCT = 0x3, /* (fast) for update statistics */
RESULT_TYPE::COUNT_DISTINCT 타입을 추가하여 count 연산 전용 병렬 처리 경로 제공
기존 mergeable list, row by row 와 함께 세 번째 결과 처리 방식으로 추가
2. Access Spec Flag 추가
ACCESS_SPEC_FLAG_COUNT_DISTINCT
count()/count(distinct) 최적화가 가능한 쿼리를 표시하는 플래그 추가
3. Count Distinct 체크 로직
다음 조건을 만족하는 쿼리를 parallel heap scan::COUNT_DISTINCT 으로 처리:
BUILDVALUE_PROC 타입
Sequential scan만 사용
COUNT() 또는 COUNT(column) 집계 함수만 포함
다른 join이나 서브쿼리, hq쿼리가 없는 단순 쿼리

1. Result Handler 구현
각 워커 스레드가 로컬 카운트를 수행
COUNT(*): 간단한 카운터 증가
COUNT(distinct <column>): 별도의 list file에 저장하여 중복 제거
COUNT(column): NULL이 아닌 값만 카운트
2. Finalize 처리
각 워커의 카운트 결과를 원본 aggregate list에 병합
COUNT(DISTINCT): 각 워커의 list file을 연결(qfile_connect_list)하여 전체 distinct 리스트 생성
Mutex로 동기화하여 thread-safe 보장
