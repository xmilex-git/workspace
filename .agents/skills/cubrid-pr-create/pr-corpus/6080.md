## Code changes (summary)

- Add `px_worker_manager_global` / `px_worker_manager` (new .cpp/.hpp pairs) implementing a server-wide cap on concurrently running parallel-query worker threads, with reserve/release semantics.
- Add `px_callable_task.cpp`/`.hpp`: a `cubthread::task` wrapper holding exec/retire `std::function`s plus a back-pointer to the owning worker manager for automatic slot release on retire.
- Rename system parameter `parallel_heap_scan_threads` to `max_parallel_workers` (`system_parameter.c`/`.h`), broadening its scope beyond heap scan alone.
- Hook the new global manager into server lifecycle: `boot_sr.c` calls `init()`/`destroy()` on restart/shutdown; `thread_manager.cpp` reserves `max_parallel_workers` extra threads in the pool sizing.
- Update `px_heap_scan_manager.cpp`/`.hpp` and `px_heap_scan_task.cpp`/`.hpp` to acquire/release workers through the new manager instead of an ad-hoc pool.
- Simplify `query_executor.c`'s parallel-heap-scan-eligibility check to rely on the `ACCESS_SPEC_FLAG_NUM_PARALLEL_THREADS` hint flag instead of always falling back to the system parameter.
- Update root/cubrid/sa `CMakeLists.txt` to add the new parallel-query worker-manager sources/headers.

## PR body (verbatim)

http://jira.cubrid.org/browse/CBRD-25977

### Purpose

현행 구현([CBRD-25447](http://jira.cubrid.org/browse/CBRD-25447))에서는 서버 전체 기준으로 동시에 실행 가능한 parallel query execution task의 개수에 대한 제한이 존재하지 않습니다. 이로 인해, 다수의 세션에서 parallel query를 무제한으로 수행할 경우, 리소스 고갈이나 성능 저하 등의 예기치 않은 문제가 발생할 수 있습니다. 이를 방지하기 위해, 서버 전역에서 parallel query execution task 수에 대한 상한을 설정하고, 별도의 worker pool을 구성하여 parallel query execution 작업을 관리할 수 있는 관리자가 필요합니다.
