## PR body (verbatim)

<http://jira.cubrid.org/browse/CBRD-26100>

### Purpose

현재 구현(Parallel Heap Scan)에서는 SELECT 목록에 PLCSQL 또는 JavaSP가 포함된 경우, gather:row-by-row 방식으로 각 스레드가 row를 fetch하여 메인 스레드로 전송한 후, 메인 스레드가 end-one-iteration 단계에서 수신한 row를 기반으로 PLCSQL 또는 JavaSP를 호출하는 구조입니다.
하지만, 오작동 및 TC 불안정이 우려되어 현재 구현에서는 이를 금지하였습니다.
이 금지 사항을 해제하고, 관련 TC 안정화를 진행하겠습니다.
