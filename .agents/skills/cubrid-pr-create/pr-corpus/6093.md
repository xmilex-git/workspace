## Code changes (summary)

- Reintroduce/finalize `px_worker_manager_global` and `px_worker_manager` (new .cpp/.hpp) as the merge target for the closed PR #6080, adding a global cap plus thread-local reservation bookkeeping for parallel-query workers.
- Add `px_callable_task.cpp`/`.hpp` (new files): a `cubthread::task` wrapper carrying an exec/retire function pair and an owning-manager pointer so a task can auto-release its reserved worker slot on retire.
- Add `max_parallel_workers` system parameter (`system_parameter.c`/`.h`) governing the global worker-pool size.
- Wire the manager into server lifecycle: `boot_sr.c` inits/destroys `worker_manager_global` on restart/shutdown; `thread_manager.cpp` adds `max_parallel_workers` to total thread-pool sizing.
- Update `px_heap_scan_manager.cpp`/`.hpp` and `px_heap_scan_task.cpp`/`.hpp` to reserve/release/push tasks through the new worker manager instead of directly managing a pool.
- Update root/cubrid/sa `CMakeLists.txt` to build the new `px_worker_manager*`/`px_callable_task` sources and headers.

## PR body (verbatim)

http://jira.cubrid.org/browse/CBRD-25977

### Purpose

현행 구현([CBRD-25447](http://jira.cubrid.org/browse/CBRD-25447))에서는 서버 전체 기준으로 동시에 실행 가능한 parallel query execution task의 개수에 대한 제한이 존재하지 않습니다. 이로 인해, 다수의 세션에서 parallel query를 무제한으로 수행할 경우, 리소스 고갈이나 성능 저하 등의 예기치 않은 문제가 발생할 수 있습니다. 이를 방지하기 위해, 서버 전역에서 parallel query execution task 수에 대한 상한을 설정하고, 별도의 worker pool을 구성하여 parallel query execution 작업을 관리할 수 있는 관리자가 필요합니다.
