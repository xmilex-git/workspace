### Purpose

현재 CUBRID에서는 uncorrelated subquery parallel execution이 기본적으로 활성화되어 있습니다. 그러나 장애가 발생하거나 특정 SQL에서 이 기능으로 인해 성능이 저하되는 경우, 또는 이 기능을 제외한 다른 parallel 기능의 효과를 측정하려는 경우에는 parallel(0) 힌트를 사용해야 합니다. 하지만 parallel(0) 힌트를 사용하면 uncorrelated subquery 외의 다른 parallel 기능들도 함께 비활성화되는 문제가 있습니다. 이를 해결하기 위해 NO_PARALLEL_SUBQUERY 힌트를 도입하여, uncorrelated subquery parallel execution만 선택적으로 비활성화할 수 있도록 수정합니다.


### Implementation

NO_PARALLEL_SUBQUERY 힌트 추가
XASL_NO_PARALLEL_SUBQUERY xasl flag 추가
