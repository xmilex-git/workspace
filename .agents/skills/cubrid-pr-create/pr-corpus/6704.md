# PR 6704: [CBRD-26425] Replace bcb mutex lock into atomic_latch

- state: MERGED / base: develop / created: 2025-12-11
- diff: +994 -506, 9 files (summary based on diffstat + partial diff review; very large refactor)

## Code changes (summary)

- Develop-branch counterpart of the same CBRD-26425 work merged earlier into `feature/refactor-pgbuf` (PR #6689): replaces per-BCB `fcnt`/`latch_mode` with a packed `std::atomic<uint64_t> atomic_latch` union, manipulated via `set_latch`, `add_fcnt`, `set_latch_and_fcnt`, `set_latch_and_add_fcnt`, `set_waiter_exists`, `get_fcnt`, `get_waiter_exists`, `get_latch`, `get_impl` CAS-based inline helpers.
- Adds a new `copy_bcb()` helper (not present in the earlier branch PR) to copy BCB fields including the atomic latch when relocating/replacing a BCB.
- Adds lock-free read-only fast path (`pgbuf_lockfree_fix_ro`, `pgbuf_search_hash_chain_no_bcb_lock`, `pgbuf_lockfree_unfix_ro`) used from `pgbuf_fix_release()` for unconditional OLD_PAGE read fixes, plus a `latch_last_thread` debug field on the BCB (SERVER_MODE, non-release builds) to track the last thread to acquire the latch.
- `pgbuf_promote_read_latch_release()` rewritten around a CAS loop on the atomic latch, replacing the old mutex + direct-field read/write promotion logic; adds `PGBUF_HOLDER_STAT`-based perf-stat capture inline.
- `pgbuf_page_monitor.pg_unfix_cnt` / `fix_req_cnt` converted to `std::atomic_int`; `pgbuf_initialize()` moves from a single `memset` of `pgbuf_Pool` to explicit per-field initialization to accommodate the new non-trivial atomic members.
- `PGBUF_THREAD_HAS_PRIVATE_LRU` now reads a new `thread_p->m_is_private_lru_enabled` flag populated by `pgbuf_thread_variables_init()`, called from `thread_entry`, `server_support.c`, `session.c`, and `vacuum.c` when a thread is bound to a session/task.
- Touches an extra file (`thread_manager.hpp`) compared to PR #6689, consistent with additional wiring for thread variable initialization on develop.
- Purpose per PR body: introduce atomic latch to improve page buffer performance (reduce BCB mutex contention on hot fix/unfix paths).
