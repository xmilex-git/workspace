# PR 6323: [CBRD-26178] qexec_clear_xasl 함수 수정

- state: MERGED / base: develop / created: 2025-07-17
- diff: +19 -27, 1 files

## Code changes (summary)

- In `qexec_clear_xasl` (src/query/query_executor.c), introduced a local `xasl_node *xasl_p` iterator.
- Replaced single-node `if (xasl->aptr_list) { ... }` clears for `aptr_list`, `bptr_list`, `dptr_list`, `fptr_list`, and `scan_ptr` with `for` loops that walk the entire linked list via `->next`, calling `XASL_SET_FLAG` and recursive `qexec_clear_xasl` on every node instead of only the head.
- Applied the same head-to-loop conversion for `buildlist->eptr_list` in the `BUILDLIST_PROC` case.
- Removed the trailing "clear the next xasl node" block that recursively cleared `xasl->next`, since callers of `qexec_clear_xasl` now iterate `next`-linked xasl nodes themselves (fixes double/incorrect clearing when a node was reached through a proc-specific list like `aptr_list` rather than only via `next`).
- Fixes a segfault when a hash-join's `aptr_list` xasl node has an uncorrelated subquery chained via `next`, since `qexec_clear_xasl` previously only recursed once into the head of each list instead of clearing every linked node.

## PR body (verbatim)

<http://jira.cubrid.org/browse/CBRD-26178>

### Purpose

다음과 같은 case에서 segmentation fault가 발생한다.
```sql
drop table if exists tt1, tt2, tt3, tt4;
create table tt1 (c1 int);
create table tt2 (c1 int);
create table tt3 (c1 int);
create table tt4 (c1 int);
insert into tt1 values (1);
insert into tt2 values (1);
insert into tt3 values (1);
insert into tt4 values (1);
 
select /*+ recompile use_hash */ *
from tt1, tt2, tt3
where (select c1 from tt4) > 0 and tt1.c1 = tt2.c1 and tt2.c1 = tt3.c1;
```
이는 (select c1 from tt4) > 0 부분의 xasl을 hash join proc에서 clear할 때 qexec_clear_xasl_for_parallel_aptr을 실행하지 않고 qexec_clear_xasl을 실행하기 때문에 발생한다.
따라서, 해당 코드에 대한 수정이 필요하다.

### Implementation

qexec_clear_xasl에서, next로 연결된 xasl에 대한 clear를 실행하지 않고, 상위 xasl에서 loop문을 통해 clear을 수행하게끔 코드를 변경한다.
