### Purpose

다음 6가지 문제를 해결합니다.
1. scan_open_parallel_heap_scan() 호출 시 file_get_num_user_pages()를 사용하는데, 쿼리가 인터럽트된 경우 실패하면서 디버그 모드에서 assert()가 발생하는 문제
2. memory_mapper를 삭제할 때, malloc()으로 할당된 메모리를 db_private_free()로 해제하려 시도하면서 assert()가 발생하는 문제
3. open_list()에서 tfile_vfid를 획득하려는 시점에 인터럽트가 발생하여 tfile_vfid가 NULL임에도 불구하고 write를 시도하다가 assert()가 발생하는 문제
4. 자식 스레드에서 scan_start_scan() 호출 시, mvcc_snapshot을 동시에 획득하려 하여 assert()가 발생하는 문제
5. tempcache free list에서 tempcache entry를 꺼내 transaction list에 push할 때 여러 스레드가 동시에 접근하여 tempcache entry가 유실되는 문제
6. 메인 스레드가 자식 스레드의 종료 상태를 task_ended가 아닌 scan_ended로 잘못 판단하여 메모리가 정리되지 않은 상태로 진행되고, 이로 인해 자식 스레드가 종료되지 않은 상태에서 쿼리가 종료되어 restrack()에서 assert()가 발생하는 문제

### Purpose

This update addresses the following six issues:
1. Failure in scan_open_parallel_heap_scan() when using file_get_num_user_pages()
An assert() failure occurs in debug mode if the query is interrupted during the call to scan_open_parallel_heap_scan(), which uses file_get_num_user_pages().
2. Memory deallocation mismatch with memory_mapper
An assert() failure occurs when attempting to deallocate memory allocated with malloc() using db_private_free() during the deletion of a memory_mapper.
3. assert() failure due to NULL tfile_vfid in open_list()
If an interrupt occurs at the moment open_list() attempts to acquire tfile_vfid, a write operation may still be attempted despite tfile_vfid being NULL, resulting in an assert() failure.
4. assert() failure due to concurrent access to mvcc_snapshot in child threads
When a child thread calls scan_start_scan(), it may try to access the mvcc_snapshot concurrently, leading to an assert() failure.
5. Loss of tempcache entry due to concurrent access
When a tempcache entry is removed from the free list and pushed to the transaction list, multiple threads may access it simultaneously, resulting in the loss of the tempcache entry.
6. Incorrect status check for child thread termination
The main thread misinterprets the child thread’s termination status as scan_ended instead of task_ended, proceeding without proper memory cleanup. This can cause the query to end while the child thread is still active, resulting in an assert() failure in restrack().
