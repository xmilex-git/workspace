# PR 4735: [CBRD-25037] Simplify MVCC check in heap scan

- state: MERGED / base: develop / created: 2023-10-18
- diff: +71 -44, 2 files

## Code changes (summary)

- `heap_next_internal()`: removed the local `curr_page_watcher` variable, using `scan_cache->page_watcher` directly instead; the previous copy-into-`old_page_watcher`/copy-into-`curr_page_watcher` branch for the same-VPID case was dropped so replacement into `old_page_watcher` only happens when moving to a different VPID
- Simplified end-of-function cleanup: unfix `scan_cache->page_watcher` directly instead of the old unfix-vs-replace-back-into-scan_cache branching
- `heap_scan_get_visible_version()` gained a new `forward_recdes` (peeked record) parameter, threaded through from `heap_next_internal()`
- Added a fast-path shortcut in `heap_scan_get_visible_version()`: when the peeked recdes type is `REC_HOME` and `ispeeking == PEEK`, the MVCC header is read directly from the already-peeked record; if the snapshot already indicates visibility, the function returns immediately without calling `heap_get_visible_version_internal()`, avoiding an extra page fetch/context setup
- Updated `heap_get_record_info()` call site and header declarations (`heap_file.h`) to match the simplified watcher usage and new parameter

## PR body (verbatim)

http://jira.cubrid.org/browse/CBRD-25037




- Heap scan performance in CUBRID 11.4 has declined by 56.8% when compared to CUBRID 9.3 due to operations performed to check MVCC id. Two improvements were suggested:

1. Eliminating unnecessary calls to pgbuf_replace_watcher() function in heap_next_internal() by using old_page_watcher element to cache old page watcher.

2. Optimizing the heap_next_internal() function to reduce the number of accesses to fetch the address of a single record from three times to once under specific conditions, enhancing efficiency.
