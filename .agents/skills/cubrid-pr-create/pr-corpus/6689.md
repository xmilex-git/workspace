# PR 6689: [CBRD-26425] Replace bcb mutex lock into atomic_latch

- state: MERGED / base: feature/refactor-pgbuf / created: 2025-12-05
- diff: +950 -502, 8 files (summary based on diffstat + partial diff review; very large refactor)

## Code changes (summary)

- `page_buffer.c/.h`: replaces the per-BCB `fcnt`/`latch_mode` pair with a single `std::atomic<uint64_t> atomic_latch` packed union (`latch_mode`, `waiter_exists`, `fcnt`), accessed through new inline helpers (`set_latch`, `add_fcnt`, `set_latch_and_fcnt`, `set_latch_and_add_fcnt`, `set_waiter_exists`, `get_fcnt`, `get_waiter_exists`, `get_latch`, `get_impl`) using CAS loops instead of mutex-protected fields.
- Adds a lock-free read-only fast path: `pgbuf_lockfree_fix_ro()` / `pgbuf_search_hash_chain_no_bcb_lock()` / `pgbuf_lockfree_unfix_ro()`, invoked from `pgbuf_fix_release()` for unconditional read latches on old pages, bypassing the BCB mutex entirely on hit.
- `pgbuf_promote_read_latch_release()` reworked to use CAS on the atomic latch (load/modify/compare_exchange_strong loop) instead of mutex + direct field mutation for read→write promotion.
- `pgbuf_page_monitor` counters `pg_unfix_cnt` and `fix_req_cnt` converted from plain `int` to `std::atomic_int`; increments changed from `ATOMIC_INC_32` macro to `fetch_add`.
- `PGBUF_THREAD_HAS_PRIVATE_LRU` changed to check a new thread-local `m_is_private_lru_enabled` flag instead of `private_lru_index != -1`, set up via new `pgbuf_thread_variables_init()`.
- `pgbuf_initialize()` replaces the blanket `memset(&pgbuf_Pool, 0, ...)` with explicit per-member initialization (needed because the pool now contains non-trivial C++ atomic members).
- `thread_entry.hpp/.cpp`, `server_support.c`, `session.c`, `vacuum.c` updated to call `pgbuf_thread_variables_init()` when a thread/session is bound, so the private-LRU flag and holder anchor are set once per thread.
- Overall goal: remove BCB mutex contention on the common read-fix/unfix path by moving fix-count/latch-mode bookkeeping to a lock-free atomic word.
