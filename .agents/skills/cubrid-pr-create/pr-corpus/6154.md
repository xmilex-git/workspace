# PR 6154: [CBRD-26011] Union, intersection, difference 문의 하위 select에서도 parallel heap scan이 가능하게 변경

- state: MERGED / base: feature/parallel_query / created: 2025-04-30
- diff: +1 -0, 1 files

## Code changes (summary)

- `px_heap_scan_checker.cpp`: add a `break;` for the `UNION_PROC`/`DIFFERENCE_PROC`/`INTERSECTION_PROC` cases in the proc-type switch so they no longer fall through into the `OBJFETCH_PROC`/`MERGELIST_PROC`/`UPDATE_PROC`/... branch that marks the XASL as not parallelizable.
- Net effect: subqueries under UNION/INTERSECT/EXCEPT are now eligible for parallel heap scan instead of being unconditionally rejected.

## PR body (verbatim)

<http://jira.cubrid.org/browse/CBRD-26011>

### Purpose

현재 CUBRID의 Parallel Heap Scan 기능은 UNION/DIFFERENCE/INTERSECTION 쿼리에 대해 적용되지 않습니다.
본 개선은 UNION/DIFFERENCE/INTERSECTION 쿼리에 대해서도 Parallel Heap Scan이 적용될 수 있도록 하여, 대용량 데이터 처리 성능을 향상시키는 것을 목표로 합니다.

### Implementation

_checker.cpp 모듈에서 UNION/DIFFERENCE/INTERSECTION 쿼리를 가능하게 허용합니다.
