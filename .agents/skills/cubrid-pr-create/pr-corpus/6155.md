## Code changes (summary)

- `px_heap_scan_manager.cpp`: track/reuse a `perf_monitor` per scan (`add_statistics()` accumulates stats into an existing monitor instead of always allocating a new one) and, on a non-parallel-scan `else` branch, call `qfile_reopen_list_as_append_mode()` so a partition that falls back to plain heap scan still gets an appendable list.
- `px_heap_scan_perf_monitor.cpp`/`.hpp`: add `add_statistics(scan_id, parallelism)` to merge a subsequent partition's per-thread scan stats (elapsed time, read/qualified rows) into the running totals.
- `px_list_merger.cpp`/`.hpp`: close head/tail list files before merging if already open, drop the now-redundant `last_pgptr` copy, and add a `clear()` helper; `swap_and_destroy_list_id()` now merges the two lists via `list_merger` when the original list already has tuples instead of always destroying-and-copying.
- `query_executor.c`: pass the owning `xasl` down into `qexec_init_next_partition()` so it can decide, per partition, whether to use `S_HEAP_SCAN` or `S_PARALLEL_HEAP_SCAN` and whether the list should merge results (`ACCESS_SPEC_FLAG_MERGED_LIST`) — enabling parallel heap scan across successive partitions of a partitioned table.

## PR body (verbatim)

<http://jira.cubrid.org/browse/CBRD-26012>

### Purpose

현재 CUBRID의 Parallel Heap Scan 기능은 Partitioned Table에 대해 적용되지 않습니다.
본 개선은 Partitioned Table에 대해서도 Parallel Heap Scan이 적용될 수 있도록 하여, 대용량 데이터 처리 성능을 향상시키는 것을 목표로 합니다.

### Implementation

qexec_next_scan_block() 및 qexec_init_next_partition() 에 parallel heap scan 관련 코드를 추가합니다.

### Remarks

32페이지가 넘지 않는 파티션에 대해 heap scan을 하다가, 다음 파티션은 32페이지가 넘어 parallel heap scan을 하는 경우 list가 close 되어있는 문제가 발생하여 관련 오류를 해결하였습니다.
