# PR 6352: [CBRD-26199] uncorrelated subquery parallel execution 실행 시, parallelism을 2로 고정

- state: MERGED / base: develop / created: 2025-07-30
- diff: +4 -1, 1 files

## Code changes (summary)

- In `scan_open_parallel_heap_scan` (px_heap_scan_manager.cpp), changed the unconditional `thread_p->m_px_orig_thread_entry = thread_p;` assignment to only assign when it is currently `NULL` (`if (thread_p->m_px_orig_thread_entry == NULL) { thread_p->m_px_orig_thread_entry = thread_p; }`).
- Prevents overwriting an already-set origin-thread pointer (previously set to the parent thread during parallel aptr/subquery execution) with the current thread itself, which caused a core dump related to `m_qlist_count` when a parallel heap scan started nested inside an already-running parallel subquery execution.
